# India.RUNS — Adversarial Candidate Ranking System (ACRS)

> AI-powered hiring intelligence platform that stress-tests candidate rankings using adversarial perturbations to detect manipulation and surface only the most defensible hiring decisions.

## What it does

Traditional ATS tools rank candidates by keyword match — making them trivially gameable. ACRS runs every candidate through **adversarial stress tests** (skill swaps, experience inflation, keyword stuffing, prompt injection) and measures how stable their rank is under perturbation. Only candidates whose ranking **holds up under attack** are surfaced as defensible hires.

| Feature | Description |
|---|---|
| **Adversarial Ranking** | Cosine similarity scoring + 11 configurable perturbation types |
| **Robustness Score** | Measures rank stability across N adversarial variants |
| **Manipulation Detection** | Keyword stuffing, prompt injection, experience claim validation |
| **Explainability Modal** | Per-candidate breakdown across Profile, Match, Adversarial Tests, Red Flags, Verdict |
| **Admin Panel** | Configure perturbation intensity, variant count, enable/disable attack types |
| **Analytics** | Score distribution, manipulation breakdown, trend charts |

## Getting Started

### Prerequisites

- Node.js 20+
- pnpm 9+
- PostgreSQL database

### 1. Install dependencies

```bash
pnpm install
```

### 2. Set environment variables

```bash
# .env
DATABASE_URL=postgresql://user:password@localhost:5432/acrs
SESSION_SECRET=your-secret-key-min-32-chars
```

### 3. Push the database schema

```bash
pnpm --filter @workspace/db run push
```

### 4. Seed the demo user

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"demo@acrs.io","password":"demo1234","name":"Demo User"}'
```

### 5. Run the development servers

```bash
# Terminal 1 — API server (port 8080)
pnpm --filter @workspace/api-server run dev

# Terminal 2 — Frontend (port 5173)
pnpm --filter @workspace/acrs-frontend run dev
```

Open [http://localhost:5173](http://localhost:5173) and sign in with `demo@acrs.io` / `demo1234`.

## How the Ranking Engine Works

1. **Fit Score** — Bag-of-words TF cosine similarity between candidate resume and job description
2. **Adversarial Variants** — Each candidate is run through N perturbation variants (configurable: 3–10)
3. **Rank Shift** — For each variant, the new rank is computed and the shift recorded
4. **Robustness** — `1 - (maxShift / totalCandidates)` → higher = more stable under attack
5. **Final Score** — Weighted blend of fit score and robustness score
6. **Red Flag Detection** — Keyword stuffing (TF threshold), prompt injection patterns, experience anomalies

## Perturbation Types

| Type | Description | Default |
|---|---|---|
| Skill Swap | Replace 1–3 key skills with irrelevant ones | ✅ On |
| Experience Fluff | Inflate experience by 2–10 years | ✅ On |
| Keyword Stuffing | Add 5–15 repetitions of key terms | ✅ On |
| Irrelevant Skills | Add 2–5 random unrelated skills | ✅ On |
| Experience Reduction | Reduce experience by 2–7 years | ✅ On |
| Skill Removal | Remove 1–3 important skills | ✅ On |
| Education Downgrade | Lower education level by 1–2 steps | ❌ Off |
| Education Upgrade | Raise education level artificially | ❌ Off |
| Company Reputation Change | Alter company prestige | ❌ Off |
| Award Exaggeration | Add fake awards/certifications | ❌ Off |
| Publication Falsification | Add fake research publications | ❌ Off |
